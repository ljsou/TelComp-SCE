Búsqueda por Palabras Clave

Ingrese una palabra clave (Ej,. Telco) en el "Módulo 1" para recuperar los servicios relacionados a su consulta. Los resultados serán desplegados en el "Módulo 2".

Selección de Componentes

Seleccione el componente que desee del "Módulo 2", luego arrástrelo hacia el canvas en el Entorno de Creación. Repita este procedimiento las veces que estime conveniente según sus requerimientos de composición.

Composición de Componentes

Los componentes cuentan con dos elementos que representan sus entras (In) y salidas (Out). El proceso de composición consiste, según el ejemplo en la figura, en la unión adecuada de la salida del componente "SendIMTelcoService" a la entrada del componente "PresenceTelcoService". Esta unión representa el "Flujo de Control" del Servicio en composición.

Búsqueda por Componentes Similares

Para utilizar esta funcionalidad basta con hacer "doble click" sobre cualquiera de los componentes complejos en el canvas (No aplica para los componentes de estructuras de control, ni para los de ingreso de datos). Los componentes recuperados serán desplegados en el "Módulo 3" del Entorno de Creación.

Establecimiento del Flujo de Datos 1/3

Por efectos prácticos, el orden de los componentes ha sido cambiado, ya que el "PresenceTelcoService" posee más datos de salida. Así, con el fin de establecer el "Flujo de Datos" entre estos dos componentes, basta con hacer click sobre el artista (cable de color azul) que los une. La accrión anterior desplegará un panel donde se encuentran los datos de salida y de entrada de los componentes "PresenceTelcoService" y "SendIMTelcoService" respectivamente. El desarrollador debe ser muy cuidadoso en esta parte para garantizar un "Flujo de Datos" adecuado.

Establecimiento del Flujo de Datos 2/3

En el "Paso A" el usuario selecciona la fila que desee, la cual representa el primer dato de salida del componente "PresenceTelcoService"; luego, escoge el dato de entrada (del componente "SendIMTelcoService") más adecuado para el dato de salida seleccionado (Paso B). El "Paso C" se ejecuta si el Desarrollador requiere escoger otro dato de salida del componente "PresenceTelcoService"; para ello, basta con presionar "Ctrl + Click sobre la fila de interés". Finalmente, el "Paso D" permite seleccionar el dato de entrada pertinente para el dato de salida escogido en el "Paso C".

Establecimiento del Flujo de Datos 3/3

Una vez asiganados los datos de salida a los datos de entrada correspondientes entre "PresenceTelcoService" y "SendIMTelcoService", el sistema solicita al Desarrollodor autorización para establecer o cancelar el "Flujo de Datos" definido. Cuando concluye este proceso, la artista seleccionada cambia a color verde, indicando que ahora representa una arista de "Flujo de Datos".

Ingreso de datos de usuario

En el "Módulo 7" existen 2 componentes que permiten al Desarollador ingresar datos a los servicios(componentes) que lo requieran, estos son: "User_Data" y "Data_Module". Para su uso se recomienda escribir primero el dato que se desea ingresar y posteriormente contectar el terminal de salida a la entrada del componente que se estime conveniente. Nota: Una vez el Desarrollador establezca el flujo de datos entre "User_Data / Data_Module" y cualquier otro componente, la arista que los conecta se tornará amarilla.

Flujo de Control/Ejecución

El "Módulo 7" también ofrece componentes de estructura de control, como: "Start, End, AND-Split y AND-Join", los cuales permiten configurar el "Flujo de Control o de Ejecución" del servicio en composición. En la Figura se puede observar un ejemplo de un servicio compuesto, constituido por los diferentes componentes explicados. El botón "Export" transforma el servicio creado por el Dasorrallor en un grafo en formato JSON.
